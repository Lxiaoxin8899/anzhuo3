# 快速研发系统页面布局与操作流程审查报告

审查对象：当前前台 Android 应用 `com.example.quickdev/com.example.smartdosing.MainActivity`

审查方式：ADB 读取当前前台应用、截图、UI 层级导出，并按用户路径点击体验。未再启动 `com.example.smartdosing`。

截图证据目录：`.gstack/design-reports/quickdev-20260521/screenshots/`

## 总体结论

当前界面已经是“快速研发系统”的研发工作台方向，首页、实验中心、档案库、设置页的主结构基本一致，适合平板横屏使用。

但主流程存在两个阻断级问题：实验中心任务列表返回 HTTP 500，配方库触发 Room 数据库迁移异常。这两个问题会让用户无法完成“领任务、查配方、启动实验配置”的核心闭环。视觉层面最大混淆点是左侧导航顶部仍显示 `Smart`，会把当前项目和旧 SmartDosing/投料系统关联起来。

## 审查路径

1. 首页：实验室控制台
2. 首页快捷入口：实验中心
3. 实验中心：研发任务页签
4. 实验中心：配方库页签
5. 档案库
6. 系统设置

## 主要发现

### 1. 主流程阻断：研发任务加载失败

证据：`03-lab-after-wait.png`

现象：进入“实验中心 -> 研发任务”后，页面先长时间显示加载态，随后展示 `加载任务失败：HTTP 500 Internal Server Error`。

影响：用户无法看到待领用任务，也无法继续“领取任务 -> 实验配置”的核心路径。

建议：
- 后端或本地 Web 服务返回 500 时，前端应展示中文可行动提示，例如“任务服务异常，请检查无线传输服务或点击重试”。
- 错误状态应保留“重试”和“返回首页/查看配方库”的替代动作。
- 需要从服务端日志定位 500 根因。

### 2. 主流程阻断：配方库数据库迁移异常

证据：`04-recipe-tab.png`

现象：切到“配方库”后，页面显示原始英文异常：
`A migration from 7 to 6 was required but not found...`

影响：配方库无法使用。更严重的是，这是开发者异常原文，普通用户无法判断该怎么处理。

建议：
- 修复 Room schema 版本和迁移链，避免从高版本数据库回退到低版本 schema。
- UI 层捕获数据库迁移错误时，显示中文降级提示。
- 如果这是测试设备旧数据导致，应提供“清理本机测试数据/重新初始化”的受控入口。

### 3. 品牌混淆：左侧导航顶部仍显示 Smart

证据：`01-current.png`

现象：当前 App 包名为 `com.example.quickdev`，业务名为“快速研发系统”，但左侧导航顶部仍显示 `Smart`，图标无明确“快研”或“快速研发系统”标识。

影响：用户会误以为当前程序仍是旧 SmartDosing/投料系统分支。

已处理：代码中已将外显文案改为 `快研`，可访问性描述改为 `快速研发系统`。

### 4. 首页信息架构清晰，但右侧监控区视觉权重偏重

证据：`01-current.png`

现象：首页主体是“指标卡 + 快捷操作 + 最近实验成果”，右侧系统监控占据独立列，视觉重量接近主操作区。

影响：新用户第一眼可能先读右侧设备状态，而不是进入实验中心。

建议：
- 保留右侧监控，但降低卡片边框和状态标签的视觉权重。
- “实验中心”主卡已经突出，建议继续把它作为唯一强主操作。

### 5. 设置页可扫读性较好，但列表卡片过大

证据：`06-settings.png`、`07-settings-lower.png`、`08-settings-about.png`

现象：设置项字号和间距适合触控，但每个区块高度较高，横屏下一屏只能看到少量设置组。

影响：设置项多时查找效率一般。

建议：
- 设置页可增加顶部分类锚点或左侧二级目录。
- 对高级设置区采用更紧凑的列表密度。

## 优先级建议

1. 立即修复 Room 迁移异常和任务接口 500。这是主流程阻断。
2. 清理当前 App 的外显旧品牌文案，避免和旧投料系统混淆。
3. 给错误页增加中文说明和替代动作。
4. 优化设置页密度，减少长列表查找成本。
