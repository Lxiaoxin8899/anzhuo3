# 认证机制切换说明

## 变更内容

已将局域网传输接口的认证机制**临时禁用**，方便后台系统快速对接。

### 修改的文件

1. **WebServerManager.kt** (行 153-198)
   - 注释掉了 API Key 认证拦截器
   - 所有 `/api` 接口现在无需携带 `X-API-Key` 请求头

2. **TaskReceiver.kt** (行 145-158)
   - 移除了发送端授权检查
   - 未知发送端会自动注册到授权列表

### 使用方式

#### 之前（需要认证）
```bash
# 1. 先配对获取 API Key
curl -X POST http://192.168.1.50:8080/api/device/pair ...

# 2. 携带 API Key 下发任务
curl -X POST http://192.168.1.50:8080/api/transfer/task \
  -H "X-API-Key: xxx" \
  -d '{...}'
```

#### 现在（无需认证）
```bash
# 直接下发任务，无需配对
curl -X POST http://192.168.1.50:8080/api/transfer/task \
  -H "Content-Type: application/json" \
  -d '{
    "schemaVersion": "1.0",
    "transferId": "TF-1707350400000-TEST",
    "senderUID": "BACKEND-001",
    "senderName": "研发管理系统",
    "timestamp": 1707350400000,
    "task": {...},
    "recipe": {...}
  }'
```

### 测试方法

运行测试脚本验证：
```bash
cd test-scripts
chmod +x test-no-auth.sh
./test-no-auth.sh
```

### 安全提示

⚠️ **注意**：无认证模式仅适用于开发/测试环境或可信的局域网环境。

**生产环境建议**：
- 重新启用认证机制（恢复原代码）
- 或者添加 IP 白名单限制
- 或者在设置中添加"信任模式"开关

### 如何恢复认证

如果需要恢复认证机制：

```bash
# 回退到上一个版本
git checkout c18be0c -- app/src/main/java/com/example/smartdosing/web/WebServerManager.kt
git checkout c18be0c -- app/src/main/java/com/example/smartdosing/data/transfer/TaskReceiver.kt

# 重新编译
./gradlew installDebug
```

或者参考 `局域网传输401错误排查指南.md` 让后台团队实现配对流程。
