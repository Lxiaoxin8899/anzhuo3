# 任务中心闭环整改说明

## 更新摘要
- “开始投料”模块已正式下线，所有按钮、导航和文案统一指向“研发配置”。
- 研发流程闭环更新为：Web 发布 → 任务中心 → 研发配置 → 配置记录 → 投料记录（只读归档）。
- Web/IDE 插件关联的调试脚本已清理，避免误触旧流程。

## 新的研发配置闭环
1. **Web 发布**：后台按配方/批次发布研发任务，包含配方、客户、负责人等字段。
2. **任务中心**：终端领取任务、查看设备状态，任一“开始配置”都会打开 `material_configuration/...` 路由。
3. **研发配置**：统一的材料配置界面支持导入 CSV/任务重做/记录纠错，保存后写入配置记录仓库。
4. **配置记录**：以任务/客户等维度回溯研发输出，可再次发起“研发配置”或纠错。
5. **投料记录（只读）**：保留历史生产数据，仅供查询/比对，不再作为入口。

## 导航与入口约束
- 底部导航仅保留：首页、任务中心、配方管理、记录、设置；所有“投料”入口改为“研发配置”。
- `SmartDosingNavHost` 提供 `navigateToMaterialConfiguration()` 扩展，禁止再拼接 DOSING 路由。
- `HomeScreen`、`RecipesScreen`、`RecordsScreen` 等 UI 统一使用“研发配置”“导入研发任务”等文案。
- 配置记录详情、任务中心、记录卡片等回跳操作均传入 `recordId`/`taskId`，保证配置数据闭环。

## 后续建议
1. 为研发配置写入流程补充进度提示与失败重试，方便操作员确认是否写入成功。
2. 配置记录与任务中心打通状态同步，例如任务完成后自动更新为“研发配置完成”。
3. 将投料记录页标题改为“生产投料（只读）”，进一步降低误解概率。
4. 监控/日志中新增 `material_configuration` 关键字，方便排查仍旧访问 DOSING 路由的残留调用。
