# SmartDosing 实验室风格结构调整方案

## 1. 导航架构优化 (Navigation Hierarchy)
### 核心思路：从“功能导向”转为“流程导向”
- **首页 (Dashboard)**：作为研发状态的“中央控制室”，展示实时天平状态、当前进行中的实验任务、以及今日实验汇总。
- **实验中心 (Lab Center)**：合并“任务中心”与“配方管理”，方便用户在选择配方和执行任务之间快速切换。
- **档案库 (Archive)**：集中管理历史记录和统计报表。
- **系统设置**：保持独立，集成设备管理。

## 2. 页面内容重组 (Page Content Reorganization)
### 首页 (HomeScreen -> DashboardScreen)
- **状态栏 (Status Bar)**：在顶部显示蓝牙天平、TTS、Web 服务的连接状态。
- **进行中任务卡片**：突出显示当前未完成的实验，提供“继续实验”一键入口。
- **数据简报**：以图表或指标卡形式展示研发精度分布、今日投料总量等。

### 实验中心 (New LabCenter)
- 采用双标签页（Tab）设计：
    - **待办任务**：从系统同步的生产/研发指令。
    - **配方库**：可直接调用的实验室配方。

### 投料作业页 (MaterialConfigurationScreen)
- **大屏自适应优化**：针对实验室平板，采用“左侧清单 + 右侧主控”的布局。
- **常驻标题栏**：在投料过程中始终可见配方名称、目标总重、当前实时总重。

## 3. 技术实现要点
- **状态共享**：通过全局 `ViewModel` 或 `CompositionLocal` 共享天平连接状态，确保所有页面都能感知硬件状态。
- **响应式布局**：强化 `LargeScreenLayout`，在 Tablet 模式下利用更多横向空间展示实验参数。

## 4. 实施路线图
1. **Phase 1**: 修改导航定义，建立 Dashboard 基础框架。
2. **Phase 2**: 重构首页，实现状态监控与任务恢复逻辑。
3. **Phase 3**: 优化投料作业页的布局，提升专业感。
